> Synced against live code at commit `d69686e` (resilient-storm) on 2026-07-06 — a full
> read-only audit of `main` plus the working tree. §9 is the prioritised findings register

## 1. System Overview

SchemeStealer is a mobile-first Warhammer paint-matching app. A painter photographs a
model, the app detects the dominant colours, and recommends matching paints across six
brands (Citadel, Vallejo, Army Painter, AK, Pro Acryl, Two Thin Coats) with a
base → highlight/layer → shade → wash recipe per colour.

There are two user flows, sharing one engine and one recipe card:

| Flow | Purpose | Background removal | Identity |
|------|---------|--------------------|----------|
| **Miniscan** | Photograph a painted miniature | Yes (client-side) | Green "Imperial" |
| **Inspiration** | Extract a palette from any image | No (whole image) | Purple "Warp" |

**Monorepo layout**

│  ├─ main.py             ASGI entrypoint, scan endpoints, CORS, rate limiting

│  ├─ routes/             ml_data.py, analytics.py  (data-collection side channels)
│  ├─ services/           miniature_scanner.py, inspiration_scanner.py, recipe_builder.py

## 2. Deploy Topology

```
            ┌──────────────────────────┐         ┌───────────────────────────┐
  Browser ──┤  Vercel (Next.js 16 SSR)  ├──HTTPS──┤  Render (FastAPI, 1 worker)│
            └──────────────────────────┘  CORS    └───────────────┬───────────┘
                       │                                          │
              NEXT_PUBLIC_API_URL                       Supabase (service-role key)
                                                          └─ or local data/ files
```

- **Frontend:** Vercel. Minimal `next.config.ts` (Turbopack only; no security headers, no
  `middleware.ts`). `app/sitemap.ts` + `app/robots.ts` exist.
- **Backend:** Render web service, configured by `render.yaml`:
  `gunicorn main:app --workers 1 --worker-class uvicorn.workers.UvicornWorker
  --forwarded-allow-ips="*" --timeout 300`. Single worker on a ~512 MB tier.
- **GPU disabled deliberately** (`main.py`, `render.yaml` env): `CUDA_VISIBLE_DEVICES=""`,
  `OMP_NUM_THREADS=1`, `ORT_DISABLE_PROVIDERS=…` — prevents an onnxruntime device-discovery
  hang on Render's shared Linux containers.
- **Supabase env vars are NOT in `render.yaml`** — only `PYTHON_VERSION` + the GPU-off vars.
  `SUPABASE_URL`/`SUPABASE_SERVICE_KEY` must be set in the Render dashboard or every
  ML/analytics write silently falls back to the ephemeral filesystem (see §7 and §9-P0).
- **Cold start:** scanners pre-warm in a daemon thread started inside the ASGI worker via
  the `lifespan` handler (`main.py:104-108`), because threads do not survive gunicorn's
  fork. Scan requests block on a `_scanner_ready` Event (up to 250 s, `main.py:88-93`).
  The frontend masks the cold start by polling `/api/ready`.
- **Concurrency model:** scans are serialised by `asyncio.Semaphore(1)`
  (`_MAX_CONCURRENT_SCANS`, `main.py:66-67`) and run in an executor; the shared
  engine/matcher state (precomputed NumPy arrays) is read-only at request time, so the
  single-worker deployment has no request-level data races. A keep-warm GitHub Action
  (`.github/workflows/keep-warm.yml`) pings `/health` every 10 minutes during waking hours.

## 3. End-to-End Pipeline (the spine)

```
[Camera/File]                                          schemestealer-react
     │  app/miniature/page.tsx  (capture="environment")
     ▼
[Client background removal]  @imgly/background-removal  lib/backgroundRemoval.ts
     │  → RGBA PNG (miniature only; inspiration sends original)
     ▼
[useScan] hooks/useScan.ts  → lib/api.ts → lib/apiClient.ts
     │  FormData POST, AbortController, 120 s timeout
     ▼  NEXT_PUBLIC_API_URL  ───────────────────────────────────────────────────►
┌──────────────────────────────────────────────────────────────────────────────┐
│ POST /api/scan/{miniature,inspiration}   main.py                              │ python-api
│   _reject_oversize (Content-Length pre-check) → _await_scanner_ready()        │
│   file.read() → _validate_upload (MIME + 10 MB)                               │
│   Image.open (PIL magic bytes, MAX_IMAGE_PIXELS=25M) → thumbnail(1024)        │
│   miniature: RGBA required (client bg removal is mandatory for mode="mini")   │
│   Semaphore(1) → run_in_executor( scanner.scan )                              │
│        │                                                                      │
│        ▼  services/{miniature,inspiration}_scanner.py                         │
│   SchemeStealerEngine.analyze_miniature   core/schemestealer_engine.py        │
│     1 quality check + CLAHE on L* (decisions only; colours measured from      │
│       the un-enhanced truth image)       core/photo_processor.py              │
│     2 auto white balance: grey-pixel estimation + CAT16 partial adaptation    │
│       (D=0.7, linear light; skipped if <2% neutral evidence)  utils/helpers.py│
│     3 background crop via alpha bbox (mini) / full frame (inspiration)        │
│     4 resize to 300 px analysis width                                         │
│     5 extraction: SLIC superpixels → K-means (K=20) over superpixel means in  │
│       OKLab → ramp-aware merge (one painted surface = one card) → union-median│
│       cluster representatives            core/smart_color_system.py           │
│     6 per-colour recipe build  _build_recipes_with_ml_features                │
│          • classify family        core/color_engine.py  (single classifier)   │
│          • neutral subdivision    Dark/Light Grey, Off-White/White by L*      │
│          • match base/highlight/shade  matcher + recipe_geometry/recipe_graph │
│          • wash derivation        config.WashMapping / recipe_builder         │
│   → colors[] {rgb, lab, hex, %, family, position, reticle?, paintRecipe}      │
└──────────────────────────────────────────────────────────────────────────────┘
     │  JSON response  ◄───────────────────────────────────────────────────────
     ▼
[Zustand commit]  lib/store.ts  setScanResult()  (mode-guarded; discards stale result)
     │
     ▼
[Results page]  app/{miniature,inspiration}/results/page.tsx
     │  per-colour reveal animation (reticle / orb)
     ▼
[Shared recipe card]  components/shared/PaintRecipeCard.tsx
        mode="miniature" → green Imperial   |   mode="inspiration" → purple Warp
```

**Key contract:** the backend returns a structured `paintRecipe` **per brand** for every
detected colour; the frontend renders it through one shared card driven by a `mode` prop.
(The offline fallback currently breaks this contract — see §6 and §9-P1.)

## 4. Backend Module Map

| Module | Responsibility |
|--------|----------------|
| `main.py` | ASGI app, `lifespan` pre-warm, pinned-origin CORS, slowapi rate limiting, scan + feedback endpoints, `/health` + `/api/ready` probes, `/api/paints` (module-cached). |
| `core/schemestealer_engine.py` | Orchestrator. `analyze_miniature()` runs quality → AWB → crop → resize → extract → recipe build. **Requires `precomputed_rgba` for `mode="mini"`.** Families are re-derived from each paint's own LAB via `classify_family` at load (drifted DB values overridden, `:107-116`). Neutral subdivision at `:337-350`. Hard monotonicity check `_monotonic_ok` for curated graph edges at `:430-437`. |
| `core/photo_processor.py` | Quality assessment (blur/glare/exposure/colour-cast) and CLAHE enhancement applied to the L* channel only. Dual-image discipline: enhanced image steers decisions, colours are measured from the untouched original. |
| `core/smart_color_system.py` | Dominant-colour extraction: SLIC superpixels (~40 px/segment) → K-means (`OVERSEGMENT_K = 20`) over superpixel means in OKLab → order-independent complete-linkage merge with a **ramp-aware metric** (chroma-dependent L-discount: neutrals keep lightness identity, chromatic ramps collapse to one card) → **union-median representative over real pixels** with a base-coat bias for dark neutral clusters (`:440-492`). Specular trim via joint L/C mask. |
| `core/color_engine.py` | **Single colour-family classifier** `classify_family()`: nearest LAB exemplar by CIEDE2000 over `color_anchors.json`, metallic flag path (gold/silver/bronze + vivid fallback), one hard threshold (black floor L<10). Also the `PaintMatcher`: OKLab shortlist-48 → CIEDE2000 rank, family gate + adjacency, ΔE ceiling 30, metallic per-metal ΔE competition (`METALLIC_WIN_TOLERANCE = {gold: 4.0, bronze: 4.0, silver: 1.5}`, `:381-382`), same-brand dedup. |
| `core/recipe_geometry.py` | Highlight/shade derivation in **OKLCH with a single soft-scored objective** — no tier ladder. Continuous hue field (warm pole ~110°, cool ~195°), headroom-scaled ΔL, chroma ramp, **soft monotonicity hinge** `_monotonicity_penalty` (`:246`), opacity/vibrancy tie-breakers; honest-empty above a score threshold (`derive_partner`, `:303-352`). |
| `core/recipe_graph.py` | Curated official base→highlight/shade chains; authoritative when present. Graph edges are accepted only if they pass the engine's hard monotonicity check, else geometry derives a partner. |
| `core/colour_maths.py` | Shared `ciede2000_single()` (skimage), OKLab/OKLCH transforms, CAM16-UCS (implemented + validated but deliberately not wired into ranking — benchmark tied with CIEDE2000), circular hue mean. |
| `services/{miniature,inspiration}_scanner.py` | Thin wrappers: call the engine, format the API response (including the composited reticle JPEG for Miniscan), load the wash DB. |
| `services/recipe_builder.py` | Assembles the per-brand recipe; wash via a guaranteed-fill archetype ladder (own brand → universal dark → cross-brand donor). |
| `routes/ml_data.py` | `/api/ml/*` — scan/colour/behaviour/feedback logging to Supabase or local files. |
| `routes/analytics.py` | `/api/analytics/*` — page/scan/ko-fi events to Supabase or daily CSV. |
| `utils/supabase_client.py` | Lazy Supabase client from `SUPABASE_URL` + `SUPABASE_SERVICE_KEY`; absent → silent file fallback. |
| `utils/auth.py` | `require_admin_key` — guards aggregate reads. **Fail-closed on Render** (403 until `ANALYTICS_ADMIN_KEY` is set in the dashboard); open only in local dev. |
| `utils/helpers.py` | Auto white balance: near-neutral (grey-pixel) illuminant estimation + CAT16 partial adaptation D=0.7 in linear light; skips when <2% neutral evidence. The old shades-of-grey design is gone (it injected ~10 ΔE00 on clean paint photos). |

## 5. Colour-Science Core

The project enforces two hard invariants and they are upheld:

1. **One colour-family classifier.** `color_engine.classify_family()` is the single
   source of truth; the frontend mirror (`lib/offlineColorDetection.ts` →
   `lib/colorAnchors.ts`) is generated from the same exemplar set, so the two cannot
   silently diverge. DB families and detected families are produced by the same logic
   (0 classifier disagreements in the current harness). No HSV hue bands exist anywhere.
2. **One rgb→lab.** Backend uses skimage `rgb2lab` (D65); the frontend
   (`lib/colorConversion.ts`) deliberately hard-codes skimage's exact `xyz_from_rgb`
   coefficients and D65 white point so the conversions agree.

**Metric policy:** OKLab Euclidean = geometry (K-means embedding, matcher retrieval
shortlist, recipe targets); CIEDE2000 = final ranking, badges, and the classifier.
CAM16-UCS is implemented in `colour_maths` but intentionally unwired (no measured benefit).

**Matcher contract** (`core/color_engine.py` `PaintMatcher`): family gate (detected family
+ adjacency) → ΔE ceiling (`BASE_MATCH_DELTA_E_CEILING = 30`) → metallic handling → same-brand
dedup within ΔE 2.0. Metallic is no longer a hard binary gate: each metal competes per-brand
under `METALLIC_WIN_TOLERANCE` (gold/bronze 4.0, silver 1.5 — silvers are colorimetrically
mid-greys), and the display family relabels to a metal only on a unanimous metallic win
across brands. An empty in-family pool returns `None` → "No match found", never a far-off
cross-family paint.

**Recipe derivation** (`core/recipe_geometry.py`): rebuilt in OKLCH around one
soft-scored objective. `target_lab()` lightens/darkens along a headroom-scaled ΔL with a
continuous hue rotation toward the warm (yellow) or cool (cyan) pole and a chroma ramp;
candidate selection combines OKLab distance, a soft family penalty, a **soft monotonicity
hinge** (a highlight darker than its base is heavily penalised, not forbidden), and
opacity/vibrancy tie-breakers, returning an honest empty above a score threshold. The
**hard** monotonicity guard survives only where curated `recipe_graph` edges are accepted
(`schemestealer_engine.py:430-437`): a stored edge that would invert lightness is rejected
and geometry derives the partner instead — so served recipes can still never invert.

**Ground truth:** `paints_groundtruth.json` (1,312 paints, six brands, Scale75 dropped) with
swatch-median measured LAB for all 1,216 opaque paints (`color_source: swatch-median`;
96 washes/inks remain `assumed`) and a `swatch_spread` metallic signature field (stored,
not yet consumed). `color_anchors.json` is built from DB medoids + curated seeds by
`scripts/build_color_anchors.py`, which also regenerates `lib/colorAnchors.ts` for parity.

**ΔE note:** the backend computes CIEDE2000 via skimage; the frontend ships an
*independent* manual CIEDE2000 (`lib/deltaE.ts`). They are held together by shared golden
fixtures (`python-api/tests/color_family_fixtures.json`, ~450 cases, run on both sides) and
`tests/fixtures/parity.json` — two implementations, one contract.

**Measured accuracy (harness, frozen 2026-07-03 in `python-api/baseline_harness.json`):**
synthetic-sweep hard-miss 9.4% (203/2,160), 0 classifier disagreements, no missing
families; clustering recovery 1.0 (incl. small trims); distortion-stable on clean/noise/
JPEG, degrades under strong white-balance/exposure casts (underexposed 0.30 — accepted as
physics, the paint genuinely looks different). Regenerate reports with
`python-api/run_all.py` (or `run_all_tests.ps1 -Harness`).

## 6. Frontend Architecture

**App Router** (`schemestealer-react/app/`) — Next 16.1.4, React 19.2.3, Tailwind 4,
Zustand 5.

```
/                        home → ModeSelector
/miniature               upload + camera (warm-up gate via useApiReady)
/miniature/results       reticle reveals + PaintRecipeCard per colour
/inspiration             Warp Portal upload
/inspiration/results     colour orbs + PaintRecipeCard per colour
/forge                   Inventory (HexGrid), Forge Mix (Alchemy Crucible), Requisition Cart
/convert/[conversionSlug]  programmatic SEO conversion pages — LIVE (static params,
                           FAQ JSON-LD, OG images, SwatchCompare/DeltaEBadge)
/paints/[brand]/[slug]   programmatic SEO paint pages — LIVE since the Phase 0 commits
                         (SwatchCompare/DeltaEBadge/CopyHexBadge, in the sitemap)
```

`app/sitemap.ts` covers home/miniature/inspiration/forge + `/convert/...` + `/paints/...`
(added in the Phase 0 commits); `app/robots.ts` allows all, disallows `/api/`.
`app/error.tsx` and `app/not-found.tsx` exist. Navigation is a fixed bottom bar; linear
`router.push()` only. Both results pages are refresh-safe (fall back to persisted
`scanHistory`, then a themed empty state — never a 404). `app/manifest.ts` + next-pwa are
wired and `public/icons/` ships generated placeholder icons (2026-07-07) — real brand art
still wanted.

**Zustand store** (`lib/store.ts`)

State: `currentMode`, `currentScan`, `scanHistory` (last 10, persisted), `cart`,
`inventory`, `customMixes`, `isScanning`/`isLoading`, `error`, `offlineMode`,
`preferredBrands`, `preferredRegion`.
Notable hygiene:
- `setScanResult()` **discards a result whose `mode` no longer matches `currentMode`** —
  the core guard against mode-switch races (covered by `store.test.ts`).
- Blob object URLs are revoked before replacement and on clear; `toPersistableScan()`
  strips `imageUrl`/`imageData`/reticles before writing to localStorage.
- `QuotaExceededError` is caught and history cleared/retried.

**Scan orchestration** (`hooks/useScan.ts` → `lib/api.ts` → `lib/apiClient.ts`)

`AbortController` cancels in-flight scans on unmount; `apiClient` enforces per-request
timeouts (120 s for scans to cover cold start) and raises a typed `ApiError`. On backend
failure the hook can fall back to in-browser offline detection — the offline engine and
matcher are dynamically imported to stay out of the main bundle. `healthCheck()` now
carries an 8 s abort and the no-env `API_BASE_URL` fallback is HTTPS-only (Phase 0) —
note the HTTPS default means LAN-IP device testing requires `NEXT_PUBLIC_API_URL`.

**Offline parity:** offline scans run `enhanceWithPaintRecipes` (wired in Phase 0), so
they render through the shared `PaintRecipeCard` like backend scans. The offline paint DB
(`lib/paintDatabase.ts`, generated) carries all **1,312** paints — full parity with the
backend DB.

**Cold-start gate** (`hooks/useApiReady.ts`)

Polls `/api/ready` every 5 s, backing off to 30 s after 5 minutes, 4 s per-poll timeout,
continue-on-error. Pages render a themed "machine spirit awakening" / "warp conduit
stabilising" overlay until ready. The warm-up overlay carries a proactive
"USE LOCAL AUSPEX" button (Phase 0) that flips the store to offline mode immediately;
the same button still appears in the retryable-error state.

**Shared recipe card** (`components/shared/PaintRecipeCard.tsx`)

One component, `mode` prop drives the two visual identities (green Imperial vs purple
Warp): difficulty pill, compare toggle, brand selector (dropdown + swipe), per-step rows
(role spine + swatch + ΔE badge + owned/cart toggles), copy-recipe. **Premium brands
(AK / Pro Acryl / Two Thin Coats) are deliberately hidden** from the selector until the
premium tier ships — a documented temporary filter (`:180-191`) with re-enable
instructions; the premium-gate overlay is intentionally unreachable until then. The
backend still returns recipes for all six brands. Theme-dependent colours use inline
`style` (dynamic hex/gradients Tailwind cannot express); structural layout is Tailwind.

**The Forge** (`app/forge/` + `components/forge/`)

Interactive, three tabs: Inventory (HexGrid), Forge Mix, Requisition cart. Mixing is
**spectral.js Kubelka–Munk** (`lib/colorMath.ts:78-121`) over measured LAB with
opacity-derived pigment loads; `simulateBasecoat` models primer coverage. Users can save
custom mixes to inventory and push paints to the cart. The mixing invariant holds: Forge
output is **display-only** and never feeds detection or matching (the backend is the only
mixing authority for anything that touches recipes). No Mixbox anywhere in the dependency
tree — licence clean.

## 7. Data & Analytics Side-Channels

Both routers follow the same pattern: **Supabase if `SUPABASE_URL` + `SUPABASE_SERVICE_KEY`
are set, else local files — silently** (errors are logged, the endpoint still returns 200).

- `routes/ml_data.py` writes scan-level features, per-colour features, behaviour, and
  feedback (including an optional **`email`** field and free-text comments) to Supabase
  tables `ml_scans`/`ml_colours`/`ml_behaviour`/`ml_feedback`, or to
  `python-api/data/ml/{scans,colours,feedback,behaviour}/`.
- `routes/analytics.py` writes page/scan/ko-fi/feedback events to `analytics_events` or to
  daily `python-api/data/analytics/events_YYYY-MM-DD.csv`.
- The file-fallback writes are **synchronous disk I/O inside `async` handlers** — they
  stall the single event loop when Supabase is absent (§9-P1).
- Render's filesystem is **ephemeral**: anything written under `data/` since the last
  deploy is lost on the next one. Supabase must be configured for durability (§9-P0).
- Uploaded **images are never persisted** — read into memory, thumbnailed, analysed, then
  `image = None; contents = None; gc.collect()` in a `finally`. Only derived colour data
  (rgb/lab/hex/family/etc.) leaves the request.

The frontend logs via `lib/mlDataLogger.ts` / `lib/analytics.ts` / `useAnalytics.ts`,
keyed by a client-generated `session_id`, behind a consent banner. First-party only —
no third-party pixels.

  frontend vitest (parity fixtures + store races + colour maths) plus Playwright specs.
  Canonical local command: `.\run_all_tests.ps1` (requires PowerShell 7 — the script's
  UTF-8 em dashes break Windows PowerShell 5.1 parsing). CI runs the full pytest + vitest

- **Offline fallback is a parallel, simpler implementation** of detection + recipe
  derivation; explicitly non-authoritative (and currently renders a degraded results view,
